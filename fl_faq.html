What is a torrent (BitTorrent)?

BitTorrent - the network distribution system for large files. Therefore, this protocol is intended for channels with a high data transfer rate. With an ADSL channel, you can download the .iso image of a 700 MB CD in a few hours.
Unlike other similar protocols, BitTorrent stands out because it does not have a search engine: a file with a torrent extension is created for each shared file, which contains information about the shared file. This torrent file can be distributed by any method. For example, torrent file lists can be displayed on specialized web servers, personal web pages, or e-mail.
The torrent file contains metainformation (for example, hashes, tracker address, etc.) about the shared data. The data is distributed using its own communication protocol, based on TCP / IP.
Participating in the tracker server exchange system has allowed BitTorrent programmers to include an algorithm that excludes the appearance of incomplete files. The download program will keep statistics of the segments downloaded by other users, and will always start with the rarest segment. For example, if the BitTorrent client downloads a large file, divided into 200 segments, it is not mandatory to start with segment # 1 and end with # 200. The customer will always request the segment that is the least spread. Ideally, at 200 seeder connections the file will be completely spread over the network.



How does it work?

The information owner generates the torrent file. The client, in turn, downloads the file with the .torrent extension, which contains information about the owner's address, as well as the name, size, and hash of each file in the torrent. All this is necessary to control the process and reduce the probability of downloading an empty or wrong file. The torrent file then opens with uTorrent, Azureus, BitTorrent, BitComet, Shareaza, or any other torrent client.

What does "parked account" mean?

Use this option only if you are missing a longer period. (ex: go on holiday). This option will delay the automatic deletion of the idle account. The maximum period of inactivity is 2 years, after which the account will be deleted. Usually (unparked) accounts are deleted after one year from the last access.
Once you park your account, you will have some restrictions on browsing the site, such as blocking access to the torrent page or forum.

I received warn/mute. What do I need to do?

Warn - in free translation: warning. You get the warn when you don't follow the rules. Depending on their severity, at a cumulative number of alerts, your account may be closed. Warn can be active for 1 to 8 weeks, or permanent.
The mute often replaces the "disable" function, which closes the account. When you receive mute you can no longer post on the forum, leave comments, buy from the shop or send PMs. They can only download and upload. The mute is permanent and you do not necessarily receive a warning message when it is granted.
Warn or mute users cannot send invitations.

What is the ratio?

Ratio - the ratio of the amount of information shared to that received. You can find this report in your profile. The user should aim to keep his ration at level 1 or higher. Keep in mind that a seeder cannot give speed to everyone, so each user must do everything possible to help the other participants. If you upload and the download remains 0, your ration will be "Inf.".
Note: your profile displays the summary ratio (all upload / all download) from the moment of registration on the tracker. In addition, there is the individual ration for each torrent, indicated in the peers table on the torrent description page.

What is a passkey?

The passkey consists of 32 symbols, is individual for each user of the tracker, is registered in the torrent file when it is registered on the HDD and is seen in the announce. It is convenient for dynamic IP or if multiple users are connecting from the same IP. If the tracker does not recognize you, check that the passkey is in announce; if not, then drag the torrent file again from the tracker.


What is a whitelist? Is my torrent client whitelist?

FileList uses what is called a whitelist ("allowed list") of torrent clients. Our tracker will not communicate with your torrent client unless it is in the list of allowed clients

By using this white list of customers, we can filter and remove customers who do not follow certain rules, or who have bugs or who simply send erroneous data to our site. This ensures that everyone uses a good customer and does not affect users in the swarm in any way.

Basically whitelisted customers are banned.

The client list is regularly updated and can be viewed here: Whitelist Client .

Why is an active torrent missing?

There can be only 3 answers:
( 1 ) Torrent violated internal upload rules.
( 2 ) Uploader deleted torrent.
( 3 ) The torrent became dead (0 seeders 0 leechers), in which case it is automatically deleted after 60 days.


Why did the download stop at 99%?

This error is usually due to the uploader, who has a file or torrent with errors. If the checksum of the downloaded segment does not match the value specified in the .torrent file, the client ignores that segment and downloads it again. If the uploader has a non-compliant torrent, then there is no correct copy of the problematic segment, and the download stops for everyone very close to the end.


What is the message "a piece has failed an hash check"?

BTclients check the downloaded data. If a part has been downloaded in error, it will be downloaded again. This happens to pretty much everyone, don't worry. Some clients may automatically block users from receiving incorrect data. If you do not want to receive data from them, you need to enable this feature in the client.


What does FreeLeech, Internal, 2X Upload, Sticky, and Romanian mean?

- torrents where only upload is considered.
- torrents of our internal groups: playSD/playHD/playBD etc.
- torrents where uploading counts twice
- torrents that remain at the top of the list of torrents for a while.
